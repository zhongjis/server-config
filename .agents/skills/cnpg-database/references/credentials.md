# Server-config CNPG Credential Reference

CNPG-generated app credentials are operator-managed. Do not create or store these Secrets in `flux/secrets/`.

## Generated Secret

For cluster `<app>-cnpg-cluster`, CNPG creates:

```text
<app>-cnpg-cluster-app
```

Expected keys:

```text
user
password
host
dbname
port
```

## App HelmRelease Wiring

Use `valuesFrom` in `flux/apps/base/<app>/app/HelmRelease.yaml`:

```yaml
spec:
  dependsOn:
    - name: <app>-cnpg-cluster
  valuesFrom:
    - kind: Secret
      name: <app>-cnpg-cluster-app
      valuesKey: user
      targetPath: <chart.db.user.path>
    - kind: Secret
      name: <app>-cnpg-cluster-app
      valuesKey: password
      targetPath: <chart.db.password.path>
    - kind: Secret
      name: <app>-cnpg-cluster-app
      valuesKey: host
      targetPath: <chart.db.host.path>
    - kind: Secret
      name: <app>-cnpg-cluster-app
      valuesKey: dbname
      targetPath: <chart.db.name.path>
    - kind: Secret
      name: <app>-cnpg-cluster-app
      valuesKey: port
      targetPath: <chart.db.port.path>
```

Match `targetPath` to the chart. Existing examples:

- `flux/apps/base/authentik/app/HelmRelease.yaml`
- `flux/apps/base/langfuse/app/HelmRelease.yaml`
- `flux/apps/base/litellm/app/HelmRelease.yaml`
- `flux/apps/base/n8n/app/HelmRelease.yaml`

## SOPS Secrets Boundary

Use `flux/secrets/production/<app>-secrets-flux.yaml` only for app input secrets not generated by CNPG.

Never commit plaintext Secrets. Never add CNPG-generated `<app>-cnpg-cluster-app` Secrets to `flux/secrets/`.
